HandleProtocolError(ExceptionEventArgs)

Performs internal handling of protocol-level errors detected during terminal communication.

Implementations can override this method to provide custom recovery logic, logging, or reconnection strategies.
If the underlying exception is an System.IO.IOException, the default behavior is to close the current connection by calling Disconnect(), allowing future operations to re-establish communication. After internal handling, the error is propagated to subscribers via Verifone.ECRTerminal.ECRTerminalManager.OnProtocolError(Verifone.ECRTerminal.ExceptionEventArgs).

ShouldAllowManualAuthorization(String)

Determines whether manual authorization flows (e.g. result codes 2003, 2007) are permitted in this application.

Declaration
protected virtual bool ShouldAllowManualAuthorization(string resultCode)
Parameters
Type Name Description
System.String resultCode

The result code received from the terminal that triggered the manual prompt.
Returns
Type Description
System.Boolean

true to allow the cashier to enter an authorization code or last 4 digits and continue the transaction; false to reject the request and abort the transaction immediately.
Remarks

By default this method returns false for security reasons, since manual authorization can allow sales without online host verification. Override in a derived class to enable this flow under controlled conditions (e.g. with manager approval).
